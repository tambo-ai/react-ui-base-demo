Create 6 route group directories and layout files. Each layout is a ROOT layout (includes html + body tags) that mounts TamboProvider. This is critical for CSS isolation — without html/body, it becomes a nested layout and loses the full-page-reload behavior.

IMPORTANT: Route groups use parentheses for the directory name (e.g., `(primer)`), but the URL path comes from a nested folder. The structure is:
- `src/app/(primer)/primer/page.tsx` renders at `/primer`
- `src/app/(primer)/layout.tsx` wraps all pages in the (primer) group

Each layout follows this exact pattern (only the function name and comment differ):

```typescript
"use client";
import { TamboProvider } from "@tambo-ai/react";
import { tamboApiKey } from "@/lib/tambo";

export default function PrimerLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>
        <TamboProvider apiKey={tamboApiKey}>
          {children}
        </TamboProvider>
      </body>
    </html>
  );
}
```

Create this for all 6 skins, changing only the function name:
- `PrimerLayout` in `(primer)/layout.tsx`
- `PolarisLayout` in `(polaris)/layout.tsx`

- `NeobrutalismLayout` in `(neobrutalism)/layout.tsx`
- `NesLayout` in `(nes)/layout.tsx`
- `RetroLayout` in `(retro)/layout.tsx`

Each file MUST have `"use client"` at the top — TamboProvider uses React Context which requires a client component.
Each file MUST include `<html lang="en">` and `<body>` — this makes it a root layout, enabling CSS isolation via full page reload.
